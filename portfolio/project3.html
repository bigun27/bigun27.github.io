Third Party Warehouse Inventory System

A manufacturing client moved inventory to a commission warehouse in Mexico for distribution to customers in the region. However, the warehouse did not have a system to keep track of the case serial numbers. To receive inventory into the application, the system requires the scan of a QR code which is applied to the pallet at time of shipment from the USA. Upon scanning the code in Mexico, the inventory is received with the pallet number, item number, quantity per case and each case number. When cases leave the warehouse and are shipped to customers, the warehouse employees will scan each case number and inventory is reduced. Nightly, the inventory file is backed up and transmitted to the US office. This helps the warehouse easily and automatically communicate to the company where all of their products are at a case level.

The local frontend is written in Python using tkinter, which connects to a Dockerized-Python API server and SQLite database.
