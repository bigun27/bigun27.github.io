"What-If" Product Costing

Before this project, the client's R&D department would be asked to create a hypothetical version of an existing product with different costs for the raw materials. With no tools to complete this task effectively, the team would spend nearly an hour on each product to change the stored price values in the ERP system. This often lead to errors getting introduced into the live business data, affecting the entire company. I was tasked with developing a standalone tool for R&D to use for this task that would not risk braking anything, and speed up the process.

I built a program that uses the Apache POI library for Java to read an Excel spreadsheet exported from the ERP and creates the new costing document in their standard format. Now, it takes just a few seconds for the team to fulfill the task.
